# PalmPalm 引き継ぎドキュメント (2026-03-12)

## プロジェクト概要

AI手相占い「ぱむぱむ」のハッカソン向けアプリ。
ユーザーの物理的な揺れ（振動センサー）をリアルタイムに検知し「占いが当たっている証拠」としてLLMに解釈させ、AIが畳み掛けるリアクションで笑いを取る。

## システム構成

```
Raspberry Pi（振動センサー）
    ↓ WS /ws/sensor
FastAPI Backend (port 8000)
    ├── AgitationEngine（動揺率計算）
    ├── MockGeminiSessionManager（MOCK_MODE=true）
    ├── TwoStageSessionManager（本番、Gemini標準API）
    ├── WS /ws/frontend（フロント配信）
    ├── GET /audio/*（音声ファイル配信 StaticFiles）
    └── POST /api/audio（マイク音声受信）
         ↓
React Frontend (port 5173)
    ├── useBackendWS（WS受信: agitation_update / ai_text / ai_audio）
    ├── useVAD（マイク → VAD → POST /api/audio）
    └── 4画面: TitlePage → RulesPage → SessionPage → EndPage
```

## WSメッセージ仕様（バックエンド→フロント）

| type | フィールド | 説明 |
|---|---|---|
| `agitation_update` | `level`, `trend` | 振動率（0-100）と傾向 |
| `ai_text` | `text` | AIテキスト（チャンク配信） |
| `ai_audio` | `url` | 音声ファイルURL（例: `/audio/line1.m4a`） |

## バックエンド セッションマネージャーインターフェース

両クラスが共通で実装するメソッド:
```python
set_broadcast_callback(callback)  # WS配信コールバック登録
start_session()                   # セッション開始
send_push(level, trend)           # 振動スパイク時の割り込み応答
receive_audio(audio_bytes, mime_type)  # マイク音声受信→Gemini→配信
```

## 現在未解決のバグ: VAD初期化エラー

### 状況

セッション画面に遷移すると `useVAD` が `MicVAD.new()` を呼ぶが、
`@ricky0123/vad-web` の WASM 初期化が Vite dev 環境で失敗している可能性がある。

### これまでの対応

1. **WASM/モデルファイルを `public/` にコピー** 済み:
   - `frontend/public/vad.worklet.bundle.min.js`
   - `frontend/public/silero_vad_v5.onnx`
   - `frontend/public/ort-wasm-simd-threaded.wasm`
   - `frontend/public/ort-wasm-simd-threaded.jsep.wasm`

2. **`vite.config.js`** に追加:
   ```js
   optimizeDeps: { exclude: ['onnxruntime-web'] }
   ```

3. **`useVAD.js`** で動的 `import()` に変更（アプリ全体クラッシュ防止）:
   ```js
   import('@ricky0123/vad-web').then(({ MicVAD }) => MicVAD.new({...}))
   ```

4. **`MicVAD.new()` オプション**:
   ```js
   workersOptions: { url: '/vad.worklet.bundle.min.js' },
   modelURL: '/silero_vad_v5.onnx',
   ```

### 次にやること（VADデバッグ）

SessionPage の `vadError` 表示または `[VAD] init error:` コンソールログの内容を確認し、
エラーメッセージに応じて対応する。

**よくある原因と対処:**

| エラー | 対処 |
|---|---|
| `no available backend` / WASM fetch失敗 | wasmPaths 設定 or onnxruntime-web バージョン確認 |
| `Permission denied` / マイク拒否 | ブラウザのマイク許可を確認 |
| `AudioWorklet` 関連エラー | `vad.worklet.bundle.min.js` の URL 確認 |
| onnxruntime-web が CJS/ESM でミスマッチ | `vite.config.js` に `@ricky0123/vad-web` も exclude 追加 |

## 環境起動方法

```bash
docker compose up          # フロント(5173) + バックエンド(8000) 起動
docker compose build frontend  # パッケージ追加後は rebuild 必要
```

環境変数:
- `MOCK_MODE=true` → MockGeminiSessionManager 使用（デフォルト）
- `GEMINI_API_KEY` → 本番時に必要

## テスト実行

```bash
cd backend && uv run pytest tests/ -v
```

全9テスト（mock: 6, two_stage: 4）が通ること。
